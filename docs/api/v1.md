# API v1

## Principles

- API は capability 単位で分ける
- モバイルはローカル更新を先に完了させる
- API は同期、アップロード、entitlement のために存在する
- すべての write API は冪等性キーを受け取る

## Authentication

- 匿名利用では `GET /v1/me` と `GET /v1/entitlement` のみ許可する
- 同期、アップロード、購入復元は認証済みユーザーのみ許可する
- 認証導入時は Clerk token を Worker で検証する

## Endpoints

### `GET /v1/me`

現在のユーザー情報、端末紐付け状態、同期可否を返す。

Example response:

```json
{
  "user": {
    "id": "usr_01",
    "auth_state": "signed_in"
  },
  "device": {
    "id": "dev_01",
    "sync_enabled": true
  }
}
```

### `GET /v1/entitlement`

現在の entitlement を返す。

Example response:

```json
{
  "state": "premium_active",
  "plan": "premium_monthly",
  "expires_at": "2026-04-09T00:00:00Z",
  "grace_until": null
}
```

### `POST /v1/uploads/request`

R2 へのアップロード許可情報を返す。

Example request:

```json
{
  "mime_type": "image/jpeg",
  "sha256": "abc123",
  "byte_size": 182736
}
```

Example response:

```json
{
  "object_key": "users/usr_01/study-items/item_01/original.jpg",
  "upload_url": "https://example-upload-url",
  "required_headers": {
    "content-type": "image/jpeg"
  }
}
```

### `POST /v1/sync/push`

ローカル更新済みの study item snapshot をサーバーへ送る。

Example request:

```json
{
  "op_id": "op_01",
  "device_id": "dev_01",
  "cursor": "client-cursor-123",
  "item": {
    "id": "item_01",
    "revision": 7,
    "title": "英単語 chapter 3",
    "image_object_key": "users/usr_01/study-items/item_01/original.jpg",
    "updated_at": "2026-03-09T10:00:00Z",
    "deleted_at": null,
    "masks": [
      {
        "id": "mask_01",
        "sort_order": 0,
        "x1": 0.1,
        "y1": 0.2,
        "x2": 0.3,
        "y2": 0.4
      }
    ]
  }
}
```

Example response:

```json
{
  "accepted": true,
  "server_revision": 7,
  "next_cursor": "srv-cursor-456"
}
```

### `GET /v1/sync/pull?cursor=...`

自端末が未取得の変更を cursor ベースで取得する。

Example response:

```json
{
  "changes": [
    {
      "type": "study_item.upsert",
      "item": {
        "id": "item_01",
        "revision": 7,
        "title": "英単語 chapter 3",
        "image_object_key": "users/usr_01/study-items/item_01/original.jpg",
        "updated_at": "2026-03-09T10:00:00Z",
        "deleted_at": null,
        "masks": []
      }
    }
  ],
  "next_cursor": "srv-cursor-789"
}
```

## Error Rules

- `409 Conflict`: revision が古い
- `401 Unauthorized`: 認証が必要
- `422 Unprocessable Entity`: payload 不正
- `503 Service Unavailable`: 一時的な再試行推奨

## Notes

- 削除は `deleted_at` を持つ tombstone として返す
- `op_id` は同一 write の再送時に必ず同じ値を使う
- 画像再アップロード回避のため、`sha256` は upload request 時に送る
