# ADR 0001: Product And Platform Direction

- Status: `Accepted`
- Date: `2026-03-08`

## Context

このプロダクトは、画像の一部を隠して想起する学習体験を中核に置く。
初期検討の時点で、以下を同時に満たす必要がある。

- オフラインでも学習が成立すること
- どんな画像でも簡単に教材化できること
- 将来の同期やバックアップに拡張できること
- 最初からサブスク導線を入れられること
- API 基盤は Cloudflare を中心にしたいこと

## Decision

以下を現時点の基本方針として採用する。

- プロダクトは `オフラインファースト` とする
- 収益モデルは `freemium + subscription-ready` とする
- バックエンドは `Cloudflare Workers + D1 + R2 + Queues + Workflows` を中心に設計する
- `Workers KV` は feature flag や設定値などの補助用途に限定する
- 認証は現時点では未導入だが、同期、バックアップ、サブスク管理が本実装に入る段階で `Clerk` を優先候補とする

## Rationale

- オフラインファーストにすることで、このアプリの使用文脈である移動中、隙間時間、試験直前に強くなる
- 画像学習ではローカル体験の速さが価値の中核であり、クラウド往復を前提にすべきではない
- サブスクは広告除去ではなく、学習資産の蓄積と保全に価値を置く方がプロダクトとの整合が高い
- Cloudflare は API、DB、オブジェクト保存、非同期処理を一貫して持てるため、バックエンド構成を比較的薄く保てる
- 認証はローカル完結の MVP では必須ではないが、複数端末同期や entitlement 管理が入る時点で必要になる

## Consequences

### Positive

- ネットワーク不通時でも主要価値を維持できる
- 将来の同期やバックアップに進みやすい
- 無料版と有料版の線引きを product と infra の両方で整理しやすい

### Negative

- ローカルとクラウドの二重状態管理が必要になる
- entitlement のキャッシュ戦略が必要になる
- 認証導入のタイミングを誤ると、データ移行が複雑になる

## Follow-Up Decisions

- 無料プランの上限設計
- entitlement モデル
- 同期モデル
- ローカル保存方式
- 認証導入タイミング
