# Product Requirements Document

## Product

- Name: `隠スタディ`
- Tagline: `隠して覚える反復想起学習アプリ`

## Product Vision

ユーザーがどんな画像でもすぐに教材化し、隠して思い出す反復想起をオフラインで回せる学習アプリを提供する。

## Problem

既存の暗記アプリはテキスト中心で、図表、地図、板書、スクリーンショットのような空間情報を含む教材には弱い。
ユーザーは画像の中の覚えたい箇所だけを隠し、視覚文脈を保ったまま想起したい。

## Target Users

- 試験勉強中の学生
- 資格試験受験者
- 語学学習者
- 図表や資料スクリーンショットで学びたい社会人

## Core Value Proposition

- 任意画像をすぐ教材化できる
- 自分で隠す行為自体が学習になる
- 画像の文脈を保ったまま想起できる
- オフラインで学習できる
- 将来は同期、バックアップ、学習履歴で継続学習を支えられる

## Product Principles

- オフラインファースト
- 学習開始までの操作を最短にする
- 画像の位置情報を壊さない
- 無料でも価値を感じられる
- 有料では継続学習の基盤を提供する

## Goals

- 初回登録から最初の学習完了までを 2 分以内にする
- オフライン時でも保存済み教材の学習成功率を 100% にする
- 学習継続と教材蓄積に課金価値を持たせる
- 無料から有料への転換を測定可能にする

## Non-Goals

- v1 での SNS 機能
- v1 での共同編集
- v1 でのデスクトップ向け高機能編集

## MVP Scope

- 画像選択
- 矩形マスク作成
- 教材保存
- 教材一覧
- 学習画面でのマスク切り替え
- オフライン利用
- 教材削除
- 既存教材の再編集
- 保存、権限、失敗時の状態表示
- サブスク導線の配置

## Subscription Strategy

### Free

- オフライン学習
- 基本的な画像登録とマスク作成
- 教材数は 20 件まで
- 同期なし
- バックアップなし
- 学習履歴は簡易表示のみ

### Premium

- 教材無制限
- クラウド同期
- バックアップと復元
- タグとフォルダ
- 学習履歴の保存と可視化
- 将来の OCR、自動マスク候補

## User Stories

- ユーザーとして、スクリーンショットを選んで重要箇所を隠したい。すぐに自分専用の教材にしたいから。
- ユーザーとして、通信がなくても学習したい。移動中や試験直前でも使いたいから。
- ユーザーとして、保存済み教材をあとから再編集したい。隠す範囲を育てたいから。
- ユーザーとして、学習資産を端末変更後も引き継ぎたい。長期で使いたいから。
- ユーザーとして、無料で試してから価値を感じた時にプレミアムへ上げたい。

## Functional Requirements

- `FR-01` ユーザーは端末内の任意画像を選択できる。
- `FR-02` ユーザーは画像上をドラッグして複数の矩形マスクを作成できる。
- `FR-03` ユーザーは作成済みマスクを削除できる。
- `FR-04` ユーザーは画像とマスク情報を教材として保存できる。
- `FR-05` ユーザーは保存済み教材の一覧を閲覧できる。
- `FR-06` ユーザーは教材詳細でマスクごとに隠す、表示を切り替えられる。
- `FR-07` ユーザーはオフライン状態でも保存済み教材の閲覧と学習ができる。
- `FR-08` ユーザーは教材削除前に確認を受けられる。
- `FR-09` アプリは保存中、失敗、権限拒否時に明確な UI を表示する。
- `FR-10` アプリは画像の表示サイズが変わってもマスク位置を正しく再現する。
- `FR-11` アプリはアプリ再起動後も学習データを復元できる。
- `FR-12` ユーザーは既存教材に対してマスクの追加、削除を再編集できる。
- `FR-13` アプリは無料、有料の entitlement 状態を保持できる。
- `FR-14` アプリは複数導線から paywall を表示できる。
- `FR-15` ユーザーはサブスクを購入、復元できる。
- `FR-16` アプリはオフライン時も直近の entitlement をキャッシュ参照できる。
- `FR-17` アプリは無料枠上限到達時にアップグレード導線を表示する。
- `FR-18` プレミアムでは同期、バックアップ、無制限保存を提供できる。
- `FR-19` 無料プランでは 20 件を超える教材保存時に paywall を表示する。
- `FR-20` プレミアム購入または復元時、ユーザーは認証済みアカウントに紐づけられる。
- `FR-21` アプリは entitlement 状態を `anonymous_free` `signed_in_free` `premium_active` `premium_grace` `premium_expired` として扱える。
- `FR-22` アプリはローカル DB に教材、マスク、学習イベント、同期キューを保持できる。
- `FR-23` 同期対象は認証済みユーザーに限定される。
- `FR-24` アプリはローカル更新を即時反映し、同期は後追いで実行できる。
- `FR-25` 教材の編集は `study item` 単位の revision を持ち、同期時に衝突判定できる。
- `FR-26` 削除は tombstone として扱い、端末間で再出現しない。
- `FR-27` 画像本体はオブジェクト保存、教材メタデータとマスクは API 経由で同期される。
- `FR-28` API は `me` `entitlement` `uploads` `sync` の能力単位で提供される。
- `FR-29` すべての write API は冪等性のために `op_id` を受け取る。
- `FR-30` pull 同期は cursor ベースで未取得変更だけを返す。

## Non-Functional Requirements

- オフラインファーストであること
- ローカルデータが単独で成立すること
- 同期は後追いで整合すること
- 保存と削除は冪等に扱えること
- 画像とマスク座標の整合性が壊れないこと
- entitlement はネットワーク不通時も一定期間参照可能であること
- 認証トークンなどの秘密情報は教材データと分離して保存すること
- 同期失敗時もローカル学習体験を止めないこと
- バックエンド API が UI 構造に直接依存しないこと

## Success Metrics

- 初回保存完了率
- 初回学習完了率
- 教材あたり平均マスク数
- 7 日継続率
- 週あたり学習セッション数
- 無料から有料への転換率
